The Laws

From the Vibe Coding Masterclass. These are your operating system for this entire build. Breaking even one will cost you days. Read once. Follow always.

Non-Negotiable 12 Laws You Will Not Break

  • 01Architecture before code. Always. For every feature, ask Claude to design the plan first. You approve it. Then code. Every single time.
  • 02One feature per prompt. Never say "build the whole pipeline." One agent, one function, one screen per prompt.
  • 03Git commit before every AI edit. Rollback in 10 seconds, always. This saves your life when the AI breaks something.
  • 04CONTEXT.md at the start of every session. The AI has zero memory. No context = hallucinations. Paste it first, every time, no exceptions.
  • 05Paste actual code, never describe it. "I have a function that does X" is not enough. Paste the function. The AI cannot misinterpret what it can see.
  • 06Constrain what must not change. Every prompt that edits existing code must say "Do NOT touch: X, Y, Z."
  • 07Fresh context for fresh tasks. After 10-15 exchanges, start a new conversation. Long contexts degrade quality dramatically.
  • 08Test every AI agent in isolation first. S-Agent, E-Agent, Evaluator — call each one in a test script before wiring them together.
  • 09Log everything during development. Agent outputs, scores, threshold decisions — log them all. Silent failures in multi-agent pipelines are catastrophic.
  • 10No API keys in code. Ever. Supabase Edge Function secrets only. .env in .gitignore from minute one.
  • 11Ask AI to verify itself. After any complex output: "Review your code. Does it reference anything not in the context provided? Fix it."
  • 12Build → Harden → Polish as separate passes. Working code first. Error handling second. Design last. Never ask for all three in one prompt.

The System 6 Agents, 3 Memory Stores, 1 Bible

Phase 1 — The Real-Time Loop (runs on every voice chunk)

🎙️
User records a voice chunk → Whisper transcribes it
Any length. Any language. Raw. Saved to Supabase Storage. Whisper transcription runs immediately on upload. Result appended to Context Window.
📊
S-Agent + E-Agent run in parallel on the full context
Input to BOTH: the entire context window (all chunks so far). S-Agent outputs: sentiment, tonality, story direction, predicted future. E-Agent outputs: all entities + user's perspective on each entity.
S-AgentE-AgentParallel
⚖️
Evaluator scores both outputs (threshold: 80)
Input: S output + E output + full context. Scores quality of understanding. If ≥ 80: tell Clarification Agent to acknowledge. If < 80: identify exactly what's missing, tell Clarification Agent what to ask.
Evaluator
💬
Clarification Agent responds to user
If score ≥ 80: warm acknowledgement + "continue your story". If score < 80: 1 specific, minimal question. Also: asks for character photo ONCE when new character introduced. Never repeats image requests.
Clarification Agent
💾
Memory stores updated → user continues → loop repeats
S store updated with latest sentiment. E store updated with latest entities. Context window extended with user chunk + clarification agent response. Loop runs for every new chunk until user clicks "Generate My Book".

Phase 2 — The Book Generation (runs when user clicks "Generate My Book")

📖
Bible assembled from 3 stores → Breaker Agent structures chapters
Bible = latest S store + latest E store + full Context Window. Breaker thinks like Premchand, Dharmveer Bharati, Nirala. Finds the most exciting, emotionally resonant chapter order. Assigns exact transcript segments to each chapter.
✍️
Proxy Writer rewrites each chapter as literary prose
40 years of experience. Has written acclaimed novels. Takes Breaker's raw chapter divisions + Bible context → produces beautifully written prose for each chapter. Preserves user's perspective and emotional truth.
🎨
Picasso Agent generates image prompts + transforms photos
For user-uploaded photos: transforms to era-appropriate style using Replicate img2img. For chapters with no photo: generates illustration from context. Can combine multiple photos. Fetches era/timing from Bible.
📚
Binder Agent assembles final PDF book
Places images into chapters at exact positions. Applies book-quality typography. Generates cover, table of contents, chapter pages. Exports final PDF. User is surprised.

Research R-02 — Sentiment + Entity Agent Design Claude.ai I'm building two parallel AI agents for real-time voice story analysis. Agent 1 — S-Agent (Sentiment Analysis): Must extract: emotional sentiment, tonality (sad/joyful/angry/nostalgic etc.), story direction (where user is steering it), political/social bias if any, user's emotional state while telling, predicted future arc of story. Agent 2 — E-Agent (Entity Extractor): Must extract: all characters (name, relationship to narrator, birth era), all events (what happened + user's perspective on it — was it good/bad?), all places (house, village, city — and what it means to the user), all time eras (1940s, 1960s, etc.), relationship graph between entities. CRITICAL nuance for E-Agent: a grandmother can be loving for one user and cruel for another. A house can feel like home or like a haunted place. The agent must capture the USER'S PERSPECTIVE on each entity, not the "objective" view. This is the most important design principle. Design for me: 1. The complete system prompt for S-Agent (in Claude API format) — it must output structured JSON only — show the exact JSON schema it should return 2. The complete system prompt for E-Agent (in Claude API format) — it must output structured JSON only — show the exact JSON schema (must include user_perspective per entity) 3. How should these agents handle incomplete information in early chunks? 4. How should entity relationships be stored as a graph structure? 5. What edge cases must each agent handle gracefully?

Vary them so they don't repeat. They must invite continuing. 2. Write 6 example clarification questions that feel like a kind friend asking, not an interrogation. What should the agent say when a user gives a very emotional, raw response? (must not be clinical or dismissive of the emotion)
